GET 방식으로 요청 후 응답페이지 받아보기

특징1. GET 방식으로 요청하는건 URL의 Header 영역에 데이터들을 포함시켜 요청함
-사용자가 입력한 값(데이터)들이 URL에 노출된다
-보안 유지가 불가능하다
-즉, 로그인이나 회원가입 같은 경우 GET 방식이 부적합하다.

특징2. Header 영역은 전송하는 길이에 제한이 있음
-방대한 데이터를 담았을 경우 초과된 데이터는 절단되어 넘어간다
-즉, 게시판 작성같은 경우 내용이 많을 수 있기 때문에 GET 방식이 부적합하다.

특징3. 장점이라고 한다면 URL에 데이터가 노출되기 때문에 해당 URL을 저장시키는 즐겨찾기 기능을 사용할 수 있다.
-즉, 검색기능 같은 경우엔 GET방식이 적합하다
